# 开发指导规范

> Agent-Skills 工作空间的开发约定（Steering 文档）。
> 版本: v0.2.0 | 最后更新: 2026-04-21

## Skill 开发流程

1. 在本工作空间根目录新建 `<skill-name>/` 目录（kebab-case 命名）
2. 按 yao-meta-skill 规范编写 `SKILL.md`（路由 + 流程骨架）
3. 创建 `agents/interface.yaml` 声明接口（输入/输出/约束）
4. 长内容放 `references/`，确定性逻辑放 `scripts/`
5. 创建 `update.sh` 定义同步到 `~/.claude/skills/` 的路径
6. 运行 `update.sh` 验证部署
7. 更新根目录 `Readme.md` 技能清单 + `CHANGELOG.md`

## SKILL.md 编写规范

- frontmatter 必须有 `name`、`description`、`metadata.author`、`metadata.version`
- `description` 字段是路由匹配的核心，须精确描述触发场景与关键词
- 流程步骤保持最小化，详细内容引用 `references/`
- 中文为主，英文为辅

## 命名规范

- Skill 目录名：kebab-case（如 `adversarial-successor-audit`）
- SKILL.md 的 `name` 字段与目录名一致
- 引用文件名：kebab-case + `.md`
- 中文文档文件名：描述性中文命名（如 `开发指导规范.md`）

## 版本管理

- 工作空间版本记录在 `CHANGELOG.md`，遵循 SemVer
- 每个 skill 独立维护 `metadata.version`
- 新增 skill → 工作空间 minor 版本 +1
- 迭代 skill → 工作空间 patch 版本 +1

## 部署规范

每个 skill 的 `update.sh` 标准模板：

```bash
#!/bin/bash
REPO_PATH="/Users/kelen/Software/github-star/Agent_Skills_kelen"
SKILL_NAME="<skill-name>"
DST="$HOME/.claude/skills/$SKILL_NAME"

if [ ! -d "$REPO_PATH/$SKILL_NAME" ]; then
  echo "源目录不存在: $REPO_PATH/$SKILL_NAME"
  exit 1
fi

mkdir -p "$DST"
rsync -av --exclude='update.sh' "$REPO_PATH/$SKILL_NAME/" "$DST/"
echo "$SKILL_NAME 已更新"
```

注意：`~/.claude/skills/<skill-name>/update.sh` 是部署端的副本，内容相同但不被 rsync 覆盖（`--exclude='update.sh'`）。

## Commit 规范

格式：`<type>: <中文简述> [en: <english>]`

type：`feat` | `fix` | `refactor` | `docs` | `chore`

示例：`feat: 新增辩证自我审查技能 [en: add dialectical-self-review skill]`
