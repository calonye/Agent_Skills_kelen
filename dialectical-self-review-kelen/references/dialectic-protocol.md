# 辩证协议详解 / Dialectic Protocol

## 五步协议

### B. 论点提取 — 怎样找到隐含假设

常见的假设藏匿点：

1. **「显而易见」陷阱**：你说「显然应该 X」的地方，往往有未经验证的假设
2. **「用户肯定想要」陷阱**：你推测的用户意图 vs 用户实际说的话
3. **「技术上只能这样」陷阱**：你以为没有替代方案的地方
4. **「这次不重要」陷阱**：你跳过的决策点（如命名、路径、默认值）
5. **「大家都这么做」陷阱**：你用惯例代替了思考

**提取技巧：** 把方案读一遍，对每个动词（「用 X」「放在 Y」「忽略 Z」）问一句「为什么不是别的？」

### C. 反驳 — 怎样有效质疑

有效反驳 vs 无效反驳：

| 无效 | 有效 |
|------|------|
| 「这可能有问题」（模糊） | 「如果用户在机器 B clone，这个路径不存在」（具体场景） |
| 「也许应该用别的方案」（无对比） | 「方案 A 要改 3 个文件，方案 B 只需改 1 个且无副作用」（量化对比） |
| 「我觉得不太好」（主观） | 「这违反了用户在第 3 条说的'不入库'要求」（引用约束） |

**反驳模板：**
```
如果 <假设> 是错的：
- 具体场景：<谁在什么时候会遇到什么问题>
- 替代方案：<另一种做法>
- 替代方案的优势：<为什么可能更好>
- 替代方案的代价：<为什么之前没选它>
```

### D. 反反驳 — 怎样诚实辩护

原则：**辩护得住就辩护，辩护不住就承认**。

辩护得住的标志：
- 能指出反驳的具体漏洞（「那个场景不存在，因为...」）
- 替代方案的代价大于收益
- 用户有明确的约束排除了替代方案

辩护不住的标志：
- 只能说「也有道理但我觉得还是...」（感觉 vs 逻辑）
- 替代方案确实更简单且无副作用
- 原方案违反了用户明确说过的要求

**诚实条款：承认错误不丢分，死撑才丢分。**

### E. 裁定 — 三种结论

- **成立**：原方案的假设/决策经过反驳后仍然站得住。注明理由。
- **推翻**：原假设被证伪，需要用替代方案。注明新方案。
- **修正**：原方向对但细节需要调整。注明调整内容。

## 何时可以跳过

- 操作完全可逆且影响范围 < 1 个文件 → 不需要辩证审查
- 用户明确说「直接做，不用想了」→ 记录但尊重
- 纯信息查询（不涉及决策）→ 不需要
